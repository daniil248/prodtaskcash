# Настройка ботов

Используются **только два бота**:

| Бот | Ссылка | Назначение |
|-----|--------|------------|
| **Пользовательский** | t.me/test_zadaniya_bot | Мини-апп для пользователей, проверка подписки/лайков (getChatMember), уведомления пользователям (sendMessage). |
| **Админский** | t.me/test_zadaniya_admin_bot | Мини-апп для админки. |

Других ботов нет. Уведомления и проверки заданий идут **только** через пользовательского бота. Если в BOT_TOKEN указать админ-бота — Telegram вернёт 403, сообщения не дойдут.

---

## Что запускать, чтобы всё работало

Нужны **три процесса** (плюс Redis, если не запущен как сервис):

| № | Что | Зачем |
|---|-----|--------|
| 1 | **Redis** | Очередь для воркера. Должен быть запущен до воркера. |
| 2 | **Бэкенд** (FastAPI) | API, проверка подписки/лайка, сайт. |
| 3 | **Воркер** (Celery) | Уведомления в Telegram (выполнил задание, вывод и т.д.), проверка заданий «Реклама» и «Приглашение». |

Без воркера не уйдут уведомления пользователям и не будут проверяться задания типа «Реклама»/«Приглашение». Проверка «Подписка»/«Лайк» идёт в самом бэкенде.

### Порядок запуска (одно окно — один процесс, либо скриптом)

```bash
# 1) Убедиться, что Redis запущен (если не сервис)
# redis-server

# 2) Бэкенд (из папки backend или корня проекта)
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 3) Воркер — в отдельном терминале (из папки backend)
celery -A app.workers.celery_app worker -Q verification
```

После смены .env перезапусти **и бэкенд, и воркер**.

### Одна команда: перезапуск всего (Docker Compose)

Если проект поднят через Docker Compose, перезапустить бэкенд, воркер и ботов одной командой:

```bash
cd /root/taskcash && bash scripts/restart-services.sh
```

Скрипт перезапускает: **backend**, **worker** (Celery), **user_bot**, **admin_bot**. Деплой из репозитория тоже вызывает этот скрипт после загрузки файлов.

## Что указать в .env (на сервере, токены не коммитить)

```env
# Пользовательский бот (test_zadaniya_bot) — мини-апп, проверки, уведомления
BOT_TOKEN=8780985171:AAG...

# Админский бот (test_zadaniya_admin_bot) — только для админ-мини-аппа
ADMIN_BOT_TOKEN=8600352425:AAF...

# Redis — нужен для воркера
REDIS_URL=redis://localhost:6379/0
```

- **USER_BOT_TOKEN** — не задавать. Всё идёт через BOT_TOKEN.
- **BOT_TOKEN** — токен **test_zadaniya_bot**. Этот же бот добавь **администратором** в каналы заданий «Подписка»/«Лайк».

### Задания «Подписка» / «Лайк»: поле «Канал»

В задании в админке в поле **«Канал (username или ссылка)»** — только **username канала**, не название.

- ✅ Правильно: `gwwfwedw7777`, `@gwwfwedw7777`, `https://t.me/gwwfwedw7777`
- ❌ Неправильно: «Тест TASK CASH» — по названию Telegram API канал не находит.

Бот (BOT_TOKEN) должен быть в канале **администратором**.

## Запуск ботов (мини-апп)

- **Пользовательский бот** (мини-апп для юзеров): `python -m bot.user_bot` — токен из BOT_TOKEN в .env.
- **Админ-бот** (админка): `python -m bot.admin_bot` — использует ADMIN_BOT_TOKEN и ADMIN_APP_URL.

После смены .env перезапусти бэкенд и воркер.

---

## Если приходят 403 при уведомлениях

В логе: `send_notification: Telegram 403 (bot cannot message user) ... BOT_TOKEN должен быть токеном пользовательского бота`.

Проверь: в .env **BOT_TOKEN** = токен именно **test_zadaniya_bot** (тот, через который открывают мини-апп). USER_BOT_TOKEN оставь пустым.
